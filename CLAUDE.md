# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Interactive mode (Cypress Test Runner)
npm test

# Run all feature files — headed Chrome
npm run full-regression-headed-chrome

# Run all feature files — headless Chrome
npm run full-regression-headless-chrome

# Run by tag
npm run login-tests-headed          # @login
npm run contact-us-tests-headed     # @contact-us
npm run smoke-pack-headed           # @smoke
npm run regression-pack-headed      # @regression and not @smoke
npm run login-and-smoke-pack        # @smoke or @login

# Run a single feature file
npx cypress run --headed --browser chrome --spec 'cypress/e2e/login.feature'

# Run by tag (custom)
npx cypress run -e TAGS="@smoke" --headed

# Generate multi-HTML report (after a run)
node cucumber-html-reports.js
```

## Architecture

This is a **Cypress BDD** project using **Cucumber** (`@badeball/cypress-cucumber-preprocessor`) with **JavaScript**. Target app: [WebdriverUniversity](https://www.webdriveruniversity.com/).

### BDD Flow

```
cypress/e2e/*.feature          →   Gherkin scenarios (Given/When/Then)
    ↓
cypress/support/step_definitions/*_steps.js   →   Step implementations
    ↓
cypress/support/page_objects/*_po.js          →   Locators & UI actions
```

### Page Object Inheritance

All POM classes extend `base_po.js`, which provides:
- `navigate(path)` — reads `baseUrl` from `cypress/fixtures/config.json` and calls `cy.visit()`
- `getPageTitle()` — returns `cy.title()`

POM files use two patterns for locators — direct `cy.get()` inline in methods, or an `element` object with getter functions. Both are valid in this project.

### Step Definitions

- `base_steps.js` — global `Before`/`After` hooks (clear localStorage) and generic steps like `I wait for {int} seconds`
- `homepage_steps.js` — navigation and home page interactions  
- `login_steps.js` — login flow steps, handles `window:alert` via `cy.stub()`
- `contactus_steps.js` — contact form steps with both fixed and parameterized data variants

### Tags

Tags control which tests run. Each `.feature` starts with `@<feature> @regression`. Available tags: `@login`, `@contact-us`, `@smoke`, `@regression`.

### Reports

Reports are written to `cypress/reports/`:
- `cucumber-html/cucumber-report.html` — single HTML (built-in preprocessor)
- `cucumber-json/cucumber-report.json` — JSON for CI
- `html-multi-report/` — rich multi-browser HTML (generated by `node cucumber-html-reports.js`)

### Config

- `cypress.config.ts` — `specPattern: "**/*.feature"`, esbuild preprocessor, no video
- `cypress/fixtures/config.json` — `baseUrl` used by all POM navigate calls
- Step definitions path: `cypress/support/step_definitions/**/*.js` (non-global, all loaded)

## Conventions

- POM files: `<name>_po.js`, class name `<name>_po`, exported as default
- Step files: `<name>_steps.js`
- Class methods use `camelCase` with prefix: `type_`, `clickOn_`, `navigateTo_`, `validate_`
- Custom Cypress commands live in `cypress/support/commands.js`
- No TypeScript in test files — only `cypress.config.ts` uses TS
